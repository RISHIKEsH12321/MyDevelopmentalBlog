APIs


AJAX

AJAX is a technique for loading data into part of a page without having to refresh the entire page. The data is often sent in a format called JavaScript Object Notation (or JSON)

Reasoning for AJAX

When a browser comes across a <script> tag, it will typically stop processing the rest of the page until it has loaded and processed that script. This is known as a synchronous processing model. With AJAX, the browser can request some data from a server and once that data has been requested, continue to load the rest of the page and process the user's interactions. This is known as asynchronous (or non-blocking) processing model.

Once a page has loaded, if you want to update what the user sees in the browser, typically you would refresh the entire page. This means the user has to wait for a whole new page to download and be rendered by the browser. With AJAX, if you only need to update a part of the page, you can just update the content of one element. This is done by intercepting an event (user clicks on link or submits a form) and requesting new content from the server via an asynchronous request


Using AJAX via JS

Basic AJAX request:
         var XHR = new XMLHttpRequest();
         XHR.onreadystatechange = function(){
            if(XHR.readyState == 4 && XHR.status == 200){
               console.log(XHR.responseText);
            }
         };
         XHR.open("GET", "https://api.github.com/zen");
         XHR.send();
